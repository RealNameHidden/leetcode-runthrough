# Retro Desk Visualizer UI Instructions

## Overall Design Concept
- **iPad Illustration**: A desk scene with:
  - A window behind the desk
  - A whiteboard on the left with handwritten problem notes/listings
  - A retro monitor on the desk displaying the interactive UI
  - General aesthetic of a cozy workspace

## Retro Monitor UI Requirements

### Visual Design
- **Retro Desktop Theme**: Windows 95/98 style, classic Mac, or other retro OS (to be decided)
- **Button Style**: Handwritten/board-style fonts (similar to Caveat, Indie Flower, etc.)
- **Color Scheme**: Authentic retro colors or adjusted for readability (to be decided)

### Expandability
- The UI should be **expandable** - users can click to maximize/expand the window beyond the monitor display
- Should smoothly transition between compact (on-monitor) and expanded (full-screen or larger) states
- Illustration should remain visible in background when contracted

### Core Functionality
The interface should display:
1. **Problem Explanations** - Clear descriptions of the algorithm/problem
2. **Interactive Visualizer** - Step-by-step animation of the algorithm
3. **Control Buttons** - Navigation between problems, step-through controls, input manipulation
4. **State Display** - Show current state (pointers, calculations, values, etc.)

### User Interactions
- Step through the algorithm (next/previous/play/pause)
- Adjust input values/array
- See real-time changes in the visualizer
- Navigate between different problems
