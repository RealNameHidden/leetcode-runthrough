# LeetCode Runthrough

This repository is structured to provide a comprehensive organization for various algorithmic categories commonly encountered on platforms like LeetCode.

## Categories:
- arrays-and-hashing
- two-pointers
- sliding-window
- stack-and-queue
- linked-list
- trees
- graphs
- dynamic-programming
- greedy
- math-and-geometry
- intervals
- backtracking
- bit-manipulation
- design-systems

Each category folder contains a .gitkeep file to ensure the folder structure is maintained in version control.

## Solutions folder

- **Location:** `solutions/`
- **Purpose:** Store organized LeetCode solutions grouped by algorithmic category (based on NeetCode-style grouping).
- **Category folders:** arrays-and-hashing, two-pointers, sliding-window, stack-and-queue, linked-list, trees, graphs, dynamic-programming, greedy, math-and-geometry, intervals, backtracking, bit-manipulation, design-systems, binary-search

Structure convention:
- `solutions/<category>/<problem-number>-<slug>/` — contains `Solution.java` and any supporting files.

Templates:
- Each category includes an `example.java` template showing the basic structure.
